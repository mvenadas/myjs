Function Declarations Vs. Expressions

Function Declaration

Function Declaration - in this type of function you will be naming the function.

function calcAge1(birthYear) {
return 2023 - birthYear;
}

age = calcAge1(1979);
console.log(age);

Note: in the "return" you can also do calculations that help to reduce the amount of code you will need to write. Also, you can call the function declaration before the function is defined. This is not possible with function expressions.

Function Expression

Function Expression - in this type of function you will not name the function, it will be an anonymous function, but you will store the function value in a variable.

const calcaAge2 = function (birthYear) {
return 2023 - birthYear;
};

const age2 = calcaAge2(1979);
console.log(age2);

Note: Both types of functions are valid, it is just the matter of preference. However, writing Function Expressions allows you to keep things more structured by defining the function first and storing it in a variable.
